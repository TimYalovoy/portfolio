# Портфолио разработчика: Примеры кода и реализованные фичи

Этот репозиторий представляет собой коллекцию рабочих примеров кода, демонстрирующих ключевые компетенции и опыт разработки в области Unity/C#. Весь представленный здесь код был написан для реальных коммерческих проектов, но адаптирован и обезличен для соблюдения соглашений о неразглашении (NDA). Из кода удалены упоминания компаний, специфичные конфигурации и приватные данные, однако сохранена суть решений и архитектурные подходы.

**Важно:** Это практические, battle-tested решения, которые использовались в продакшене, а не учебные примеры. Каждая фича решала конкретную бизнес-задачу и прошла через цикл разработки, тестирования и оптимизации.

## Цель репозитория

Этот репозиторий создан для следующих целей:

- **Создать "живую" витрину навыков** — продемонстрировать реальный опыт работы с различными аспектами разработки в Unity
- **Показать стиль кода и подходы к решению задач** — дать представление о том, как я структурирую код, выбираю архитектурные решения и работаю с производительностью
- **Облегчить процесс технического интервью** — предоставить конкретные точки для обсуждения, примеры работы с различными технологиями и демонстрацию понимания сложных концепций

## Структура репозитория

Каждая папка в корне репозитория представляет собой отдельную фичу, модуль или алгоритм. Внутри каждой папки находится реализация с подробным описанием в локальном `README.md` файле, который объясняет:

- Постановку задачи и контекст
- Выбранное решение и обоснование
- Ключевые особенности и демонстрируемые навыки
- Структуру кода и примеры использования

### Доступные фичи

- **`/Kill Process/`** — Утилита для управления процессами Unity в редакторе. Реализует механизм поиска и завершения процессов, содержащих определенные модули, что полезно для отладки и очистки окружения разработки.

- **`/Knot Detection/`** — Система детекции узлов на веревке с использованием теории узлов и Skein Relation. Реализует алгоритмы распознавания различных типов узлов (Trefoil, Figure-Eight, Square и др.) на основе анализа самопересечений физической веревки.

- **`/Mesh Deformation/`** — Система деформации мешей с физикой на основе Compute Shaders. Реализует интерактивную деформацию мешей с использованием GPU-вычислений для высокой производительности, включая пружинную физику и фильтрацию воздействий.

## Технологии и стек

Основные технологии, используемые в репозитории:

- **Unity** — игровой движок и среда разработки
- **C#** — основной язык программирования
- **Compute Shaders / HLSL** — GPU-вычисления для высокопроизводительных операций
- **Obi Physics** — физический движок для работы с веревками и мягкими телами
- **System.Diagnostics** — работа с процессами операционной системы
- **Unity Editor Tools** — создание инструментов для редактора Unity

## Как использовать этот репозиторий

1. **Просматривайте папки с фичами**, которые наиболее релевантны вашим интересам или требованиям позиции
2. **Читайте локальные `README.md` файлы** в каждой папке для полного понимания контекста, задачи и решения
3. **Изучайте код** — каждая реализация содержит комментарии и демонстрирует конкретные техники и подходы
4. **Обратите внимание:** код НЕ готов к запуску "из коробки", так как требует зависимых модулей, конфигурации Unity-проекта и внешних библиотек (например, Obi Physics). Код представлен для демонстрации подходов и решений, а не как готовый к использованию пакет.

Весь код в этом репозитории был извлечен из коммерческих проектов и адаптирован для демонстрации. Некоторые части могут быть упрощены или обобщены для сохранения конфиденциальности, но архитектурные решения и ключевые алгоритмы представлены в том виде, в котором они использовались в продакшене.
